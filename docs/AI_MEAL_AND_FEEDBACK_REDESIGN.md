# AI Meal Logging & Actionable Feedback 섹션 리디자인 전략

**목표**: 두 섹션을 일반적인 기능 블록이 아닌, 각각 다른 **제품 장면**으로 재구성  
**원칙**: 한 섹션은 좌측 중심, 다른 섹션은 중앙·구도 중심 — 레이아웃과 분위기를 구분

---

## A. Distinct Redesign Concept for Each Section

### AI Meal Logging — “입력 흐름이 한눈에”

**컨셉**: “검색 → 결과 등장 → 저장” 전 과정을 **좌측에 몰아두고**, 오른쪽에 **큰 UI**가 펼쳐지는 구도.  
- **좌측 35%**: 카피 + 짧은 서브  
- **우측 65%**: 제품 UI가 화면을 가득 채우는 느낌  

**서사**: “귀찮은 입력”이 아니라 “검색 한 번이면 끝”이라는 **즉시성·부담 해소**를 시각적으로 보여줌.

**핵심 요소**:
1. 검색창 (타이핑 또는 placeholder로 검색 행위 암시)
2. 결과 카드 — 음식명, kcal, 탄단지가 **순차 등장**
3. 저장 완료 상태 — “저장됨” 또는 체크 아이콘
4. 일일 누적 — 오늘 기록 합계 (작게 하단에)

---

### Actionable Feedback — “분석 → 결론 → 행동” 중심 구도

**컨셉**: **중앙에 큰 피드백 카드**가 있고, 주변에 요약·부족·추천이 **레이어드**로 펼쳐지는 형태.  
- **레이아웃**: 중앙 집중, 카피는 위쪽 짧게, 아래는 **피드백 UI가 메인**  
- **좌우 균형**: 추천 블록은 좌측, Coli는 우측 하단(작게)

**서사**: “숫자만 보는” 게 아니라 **“그래서 뭘 먹을지”**까지 이어지는 **실행 가능한 흐름**을 보여줌.

**핵심 요소**:
1. 오늘 요약 (3끼, kcal, 탄단지 비율) — 상단, compact
2. 부족 영양소 — “단백질 부족” 등, 시각적으로 강조
3. **추천 다음 선택** — “그릭요거트, 닭가슴살 샐러드” 등 구체적 음식명
4. 균형 바 — 보완 필요 상태

## D. Interaction and Motion Ideas

### AI Meal Logging

| 단계 | 트리거 | 모션 |
|------|--------|------|
| 1 | 섹션 in-view | 카피 fadeUp (stagger) |
| 2 | delay 400ms | 검색창 fadeIn, 커서 깜빡임 |
| 3 | delay 600ms | 타이핑 시퀀스 (닭가슴살 샐러드, 70–90ms/글자) |
| 4 | 타이핑 완료 + 400ms | 결과 카드 slideUp + scale 0.98→1 |
| 5 | + 300ms | 음식명, kcal, 탄단지 라벨 순차 opacity |
| 6 | + 400ms | 탄단지 바 width 0→목표값 (stagger 0.08s) |
| 7 | + 600ms | “저장됨” 배지 scale 0.9→1 + fadeIn |
| 8 | + 200ms | 일일 누적 strip fadeIn |

**Ambient**: 없음 — 순차 시퀀스가 끝나면 정적 유지.

**반복**: 시퀀스는 `once: true`로 한 번만. 사용자 스크롤로 다시 보이면 이미 완료 상태.

---

### Actionable Feedback

| 단계 | 트리거 | 모션 |
|------|--------|------|
| 1 | 섹션 in-view | 카피 fadeUp (stagger) |
| 2 | delay 300ms | 카드 wrapper scale 0.97→1, opacity 0→1 |
| 3 | + 200ms | 요약 라인 (3끼, kcal) fadeIn |
| 4 | + 250ms | “단백질 부족” 문장 slide from left + opacity |
| 5 | + 350ms | 추천 블록 **height 0→auto** (펼쳐지는 느낌) + border-left accent |
| 6 | + 200ms | 추천 문장 opacity, Coli scale 0.85→1 |
| 7 | + 300ms | 균형 바 width 0→72% |

**시퀀스 타이밍**: 총 1.5~1.8초.  
**강조**: 추천 블록이 **“해결책”**으로 느껴지도록 — 다른 요소보다 0.1~0.2초 늦게, 약간 더 크게 등장.

## E. Implementation Guidance

### AI Meal Logging

- **그리드**: `grid-cols-[0.35fr_0.65fr]` 또는 `[1fr_1.8fr]` — 좌측 copy, 우측 product
- **제품 영역**: `min-h-[420px] md:min-h-[500px]`, `max-w-[560px]` (우측에 여유)
- **배경**: `radial-gradient(ellipse 60% 80% at 5% 50%, rgba(127,176,105,0.1), transparent 50%), var(--bg-oat)`
- **시퀀스**: `useEffect` + `setTimeout`/`requestAnimationFrame`으로 phase 제어 (현행과 유사)
- **타이핑**: `setInterval` 또는 `for` + `await delay` — `닭가슴살 샐러드` 7글자, 70–90ms/글자

### Actionable Feedback

- **레이아웃**: 카피 `text-center`, `max-w-2xl mx-auto` / 카드 `max-w-3xl mx-auto`
- **카드**: `rounded-2xl`, `p-8 md:p-10`, `shadow-[0_24px_60px_rgba(45,90,61,0.1)]`
- **추천 블록**: `border-l-4 border-l-[var(--accent-leaf)]`, `pl-5`, 배경 `accent-leaf/5`
- **배경**: `bg-cream` + `radial-gradient(ellipse 80% 50% at 50% 100%, rgba(74,124,89,0.08), transparent 60%)`

### 공통

- **useInView**: `amount: 0.12` ~ `0.15`, `margin: "0px 0px 80px"` — 섹션 일부만 보여도 시작
- **prefers-reduced-motion**: phase를 즉시 마지막으로 하거나, 모션 duration 0.01ms

### 체크리스트

**AI Meal Logging**:
- [ ] 좌측 35% copy, 우측 65% product
- [ ] UI min-h 500px (desktop)
- [ ] 타이핑 → 결과 → 저장 → 일일 요약 순차
- [ ] 탄단지 바 width 애니메이션

**Actionable Feedback**:
- [ ] 중앙 정렬, 카드가 메인
- [ ] 요약 → 부족 → 추천 → 균형 바 순차
- [ ] 추천 블록 시각적 강조
- [ ] Coli 작고 보조

**양쪽**:
- [ ] 서로 다른 레이아웃 (좌 vs 중앙)
- [ ] 각각 다른 배경 톤
- [ ] 제품 UI 크기 확대
- [ ] reduced-motion 대응
