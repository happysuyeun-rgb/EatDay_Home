# Community + Report 섹션 리디자인 전략

**목표**: 커뮤니티와 리포트를 **연속성(continuity)** 중심 스토리로 통합하고, 감성·인사이트를 구분해 전달  
**원칙**: 게이미피케이션 없이 사회적 격려, 리포트는 개인적·유용하게

---

## A. Section Concept

### 핵심 메시지: “흐름을 이어가는”

- **Community**: 혼자가 아니라 함께 보는 것만으로도 기록이 이어짐
- **Report**: 숫자·추세가 “내가 어떻게 하고 있는지”를 보여줌
- **통합**: “사회적 응원” + “개인 인사이트”가 **하나의 연속성 경험**을 만듦

### 양면 대비

| 왼쪽 (사회·감성) | 오른쪽 (분석·인사이트) |
|------------------|------------------------|
| 흐르는 피드 | 정돈된 리포트 |
| 따뜻한 톤, 반응(좋아요·댓글) | 차분한 톤, 숫자·차트 |
| “다른 사람들도 하고 있구나” | “나는 이렇게 진행 중이구나” |
| 부드러운 색, 라운드 | 선명한 구조, 막대·차트 |

### 비게이미피케이션

- ❌ 레벨, 스트릭, 배지, 챌린지
- ✅ 공유·응원·인사이트만으로 “이어가고 싶다” 느낌

## B. Korean Copy Rewrite

### 태그라인 옵션

| 옵션 | 문구 |
|------|------|
| A | 흐름 · 연속성 |
| B | 함께 보기 · 내 흐름 |
| C | 커뮤니티 · 리포트 |

### H2 옵션

| 옵션 | 문구 |
|------|------|
| **A (연속성)** | 혼자 기록하면 끊기기 쉽고,<br/>함께 보면 조금 더 오래 갑니다 |
| **B (감성)** | 기록은 혼자서도 하고,<br/>함께 보면 더 잘 이어져요 |
| **C (실용)** | 공유로 격려받고,<br/>리포트로 내 흐름을 확인해요 |
| **D (직관)** | 같이 보면 끊기지 않고,<br/>숫자로 흐름이 보여요 |

**추천**: A (현행) 또는 B.

### 서브카피 옵션

| 옵션 | 문구 |
|------|------|
| A | 식단을 공유하고, 일일 리포트로 내 흐름을 확인해보세요. |
| B | 다른 사람의 식단을 보며 격려받고, 내 기록을 숫자로 돌아보세요. |
| C | 공유는 선택이에요. 리포트는 매일, 내 식단 흐름을 한눈에 보여드려요. |
| D | 함께 이어가는 기록과, 나만의 인사이트. |

**추천**: A 또는 C (선택·유용성 강조).

### 피드·리포트 내부 라벨

- **피드**: `오늘의 공유` / `공유 피드` / `함께 보기`
- **리포트**: `내 흐름` / `오늘 리포트` / `이번 주 요약`
- **탭**: `오늘` / `이번 주` 유지
- **인사이트 문장**: “단백질 섭취가 목표의 92%예요. 저녁에 가볍게 보충하면 좋아요.” — **개인화·실용** 톤 유지

## C. Visual Redesign Strategy

### 배경·섹션 구분

- **배경**: `bg-oat` 또는 **cream → sand** 미세 그라데이션
- **좌측 영역**: blush/lime 아주 연한 tint — 따뜻함
- **우측 영역**: forest/leaf 연한 tint — 차분함
- **구분선**: 중앙에 얇은 `border-l` 또는 **없음** (색 톤으로만 구분)

### 레이아웃

```
┌─────────────────────────────────────────────────────────────────┐
│  [태그] 흐름 · 연속성                                             │
│  혼자 기록하면 끊기기 쉽고, 함께 보면 조금 더 오래 갑니다            │
│  식단을 공유하고, 일일 리포트로 내 흐름을 확인해보세요.              │
├────────────────────────────┬────────────────────────────────────┤
│  [사회·감성]                │  [분석·인사이트]                     │
│  ┌─────────────────────┐   │  ┌─────────────────────────────┐   │
│  │ 공유 피드            │   │  │ 리포트 · 오늘 / 이번 주      │   │
│  │                     │   │  │                             │   │
│  │ [카드] 점심 샐러드   │   │  │ 오늘 균형 92%               │   │
│  │ 420kcal · ♡ 12      │   │  │ 단백질 92%... 저녁 보충     │   │
│  │                     │   │  │                             │   │
│  │ [카드] 저녁 포케     │   │  │ [탄단지 바]                 │   │
│  │ 580kcal · ♡ 8       │   │  │ [주간 차트]                 │   │
│  │                     │   │  │                             │   │
│  │ [카드] 아침 그릭     │   │  │                             │   │
│  │ 280kcal · ♡ 15      │   │  │                             │   │
│  └─────────────────────┘   │  └─────────────────────────────┘   │
└────────────────────────────┴────────────────────────────────────┘
```

### 좌측 — 사회·흐름

- **스타일**: 리스트, border-b 구분, **카드 느낌 완화**
- **피드 카드**:
  - 썸네일(또는 음식 placeholder) + 식사명 + kcal + 좋아요·댓글
  - **짧은 코멘트** 1줄: “단백질 잘 챙겼어요”, “오늘 포케” 등 (선택)
  - **featured** 라벨: “오늘의 인기” 대신 “많이 공유됨” 또는 미라벨
- **색**: blush, lime 등 부드러운 톤
- **모션**: 카드가 **위에서 아래로** 순차 등장 — 스크롤·흐름 느낌

### 우측 — 분석·인사이트

- **스타일**: **forest 다크 패널** 유지 — 이전과 다른 섹션 대비
- **탭**: 오늘 / 이번 주
- **오늘**: 균형 %, 인사이트 문장, 탄단지 바
- **이번 주**: 기록 일수, 주간 차트, 짧은 코멘트
- **인사이트 문장**: “저녁에 가볍게 보충하면 좋아요” — **실행 가능한** 톤
- **색**: white on forest, 막대는 lime/white/amber

### 시각적 차이 강화

| 요소 | 좌측 (사회) | 우측 (인사이트) |
|------|-------------|-----------------|
| 배경 | 연한 blush/lime tint | forest 다크 |
| 형태 | 리스트, 구분선 | 카드, 패널 |
| 타이포 | 중간 굵기, 이탤릭 없음 | 숫자 강조, 라벨 정돈 |
| 모션 | 상→하 흐름 | 탭 전환, 바 grow |

## D. Motion Concept

### 진입

| 요소 | 패턴 | 타이밍 |
|------|------|--------|
| 카피 | fadeUp stagger | 0 ~ 0.5s |
| 좌측 피드 | 카드별 stagger (y 16→0, opacity) | 0.15, 0.25, 0.35s |
| 우측 리포트 | scale 0.98→1, opacity 0→1 | 0.2 ~ 0.6s |
| 탭·내용 | fadeIn | 0.4 ~ 0.55s |
| CountUp | (기존) | 0.5s~ |
| 탄단지 바 | width 0→목표 | 0.55 ~ 0.9s |
| 주간 차트 | height 0→목표 | 0.6 ~ 0.95s |

### 탭 전환 (오늘 ↔ 이번 주)

- **전환**: `AnimatePresence mode="wait"` — 이전 내용 fadeOut, 다음 fadeIn
- **duration**: 0.25~0.35s
- **탭 버튼**: active 시 배경색 transition (0.2s)

### 피드 카드

- **호버**: `y: -2`, `bg` 미세 변화
- **클릭**: 없음 (정적 데모) 또는 “자세히” 등 링크(선택)

### Reduced motion

- stagger, CountUp duration 단축
- 탭 전환만 유지 (필수)

## E. Implementation Notes

### 데이터 확장 (피드)

```js
// 현재
{ meal, kcal, likes, comments, thumb, featured }

// 확장 (선택)
{ meal, kcal, likes, comments, thumb, featured, comment?: "단백질 잘 챙겼어요" }
```

### 좌측 피드

- **레이아웃**: `space-y-0` + `border-b` 또는 `space-y-3` 카드
- **스타일**: `hover:bg-black/[0.02]`, `rounded-xl`, `p-4`
- **썸네일**: `w-14 h-14 rounded-xl` (유지)

### 우측 리포트

- **패널**: `bg-[var(--accent-forest)]`, `rounded-2xl`, `p-6 md:p-8`
- **탭**: `role="tablist"`, `aria-selected` 유지
- **인사이트 문장**: `text-white/90`, `leading-relaxed`

### 배경

- **섹션**: `bg-oat` 또는 `bg-gradient-to-b from-cream to-oat/80`
- **좌측 영역**: `before` 또는 wrapper에 `rgba(232,222,213,0.3)` 좌측 gradient (선택)
- **우측 영역**: forest 패널 자체가 대비
