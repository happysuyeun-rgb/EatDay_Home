# EATDAY Hero 섹션 리디자인 전략

**목표**: 첫 화면을 프리미엄·임팩트 있게 만들고, 3초 안에 핵심 가치 전달  
**방향**: 작고 조심스러운 톤 제거 → 큰 스케일, 편집적 리듬, 제품 확신

---

## A. Redesigned Hero Strategy

### 핵심 전략
1. **스케일 리셋**: H1을 “문서 제목” 수준이 아니라 “에디토리얼 헤드라인” 수준으로 상향
2. **레이어드 제품 컴포지션**: 단일 대시보드가 아니라 2~3단 레이어(메인 카드 + 오버랩 카드 + 칩)로 깊이감
3. **컨텍스트 칩의 서사**: 칩 4개가 “기록 → 분석 → 맛집 → 리포트” 흐름을 암시
4. **Ambient 모션**: 과하지 않은 부유·페이드로 “살아있는” 느낌
5. **Coli는 보조**: 우측 하단 또는 카드 모서리에 작게, 메인 비주얼 아님

### 3초 안에 전달할 메시지
- **Easy logging**: 기록이 부담스럽지 않다
- **Smart analysis**: AI가 똑똑하게 분석한다
- **Actionable guidance**: “다음에 뭘 먹을지”까지 제안한다
- **Restaurant support**: 외식할 때도 쓸 수 있다

### 톤
- Calm but high-end (차분하면서도 고급스러운)
- Confident, not shouty (자신감 있되 과하지 않게)
- Korean wellness brand (한국형 웰니스 브랜드)

## B. Stronger Korean Copy Options

### H1 옵션 (2줄, 에디토리얼 리듬)

| 옵션 | 1줄 | 2줄 |
|------|-----|-----|
| **A (현행 강화)** | 기록은 가볍게, | 식단 관리는 더 똑똑하게 |
| **B (가치 중점)** | 입력은 줄이고, | 선택은 더 똑똑하게 |
| **C (결과 강조)** | 기록은 가볍게, | 다음 한 끼까지 이어가요 |
| **D (문제 해결)** | 식단 앱, 왜 계속 포기하셨나요? | 이번엔 다릅니다 |
| **E (감성)** | 오늘 먹은 것부터 | 다음 한 끼까지 잇는 앱 |
| **F (직관)** | 검색 한 번이면 | 칼로리·영양소·다음 추천까지 |

**추천**: A 또는 B — 짧고 기억에 남으며, “가볍게”/“똑똑하게”가 브랜드 톤과 맞음.

### 태그라인 옵션 (H1 위)

| 옵션 | 문구 |
|------|------|
| **A** | AI 식단 · 실행 피드백 · 맛집 지도 |
| **B** | AI 식단 분석 · 맛집 지도 · 일일 리포트 |
| **C** | 기록 · 분석 · 추천 · 맛집까지 |
| **D** | 식단 기록부터 다음 선택까지 |

### 서브카피 옵션 (H1 아래)

| 옵션 | 문구 |
|------|------|
| **A** | 오늘 먹은 것부터, 다음 한 끼 선택까지. 잇데이가 흐름을 잇습니다. |
| **B** | 검색 한 번으로 기록하고, AI가 부족한 영양을 짚어주고, 근처 맛집까지 연결해드려요. |
| **C** | 입력 부담은 줄이고, 실행 가능한 제안은 늘립니다. |

**추천**: A(감성) 또는 C(가치) — 너무 길지 않고, 핵심 가치를 압축.

### 상세
1. **좌측**
   - 태그: 11~12px, uppercase, leaf 색상
   - H1: 2줄, 짧은 줄(6~10자), 넉넉한 line-height
   - 서브: 1~2줄, max-width 제한
   - CTA: Primary 1개 + Secondary 1개 (링크 스타일)

2. **우측**
   - **메인 카드**: 오늘의 식단 대시보드 (kcal, 탄단지 바, 기록 요약)
   - **오버랩 카드**: 최근 기록된 식사 1건 (닭가슴살 샐러드 420kcal 등)
   - **플로팅 칩 4개**:
     - 오늘 1,240 kcal (우상단 또는 좌상단)
     - 단백질 부족 (우측 중앙)
     - 근처 고단백 맛집 3곳 (우하단)
     - AI 분석 완료 (좌하단)
   - **Coli**: 카드 모서리, w-10~12, 시각적 비중 낮게

3. **비율**
   - 데스크톱: copy 40%, product 60%
   - 모바일: 세로 스택, product 우선 노출 (order-first)

## D. Motion Plan

### 진입 시퀀스 (총 1.2~1.5초)

| 순서 | 요소 | 패턴 | 타이밍 |
|------|------|------|--------|
| 1 | 태그 | fadeUp (opacity 0→1, y 12→0) | 0 ~ 0.4s |
| 2 | H1 1줄 | fadeUp + 약한 blur(8→0) | 0.05 ~ 0.6s |
| 3 | H1 2줄 | fadeUp | 0.15 ~ 0.7s |
| 4 | 서브카피 | fadeUp | 0.3 ~ 0.75s |
| 5 | CTA | fadeUp | 0.4 ~ 0.85s |
| 6 | 메인 카드 | scale 0.96→1, opacity 0→1, y 24→0 | 0.2 ~ 0.9s |
| 7 | 오버랩 카드 | slide (x 20→0, opacity 0→1) | 0.5 ~ 0.95s |
| 8 | 칩 1~4 | stagger fadeIn + blur(6→0) | 0.55, 0.7, 0.85, 1.0s |
| 9 | Coli | scale 0.85→1, opacity 0→1 | 0.85 ~ 1.1s |
| 10 | 탄단지 바 | width 0→pct (stagger) | 0.75 ~ 1.2s |

### Ambient (진입 이후)
- **칩**: `y: [0, -4, 0]`, duration 4~5초, repeat Infinity, 각각 delay offset
- **메인 카드**: 미세한 shadow 변화 또는 없음 (가벼운 hero 유지)
- **오버랩 카드**: 호버 시 `translateY(-2px)` (선택)

### Easing
- 진입: `[0.16, 1, 0.3, 1]` (ease-out-expo 느낌)
- Ambient: `easeInOut`

### Reduced Motion
- `prefers-reduced-motion` 시 모든 애니메이션 duration → 0.01ms 또는 즉시 최종 상태

## E. Implementation Recommendation

### 타이포그래피 스케일
- **H1**: `text-[2.75rem] sm:text-6xl md:text-7xl lg:text-[4.25rem] xl:text-[4.5rem]`
  - line-height: 1.05 ~ 1.1
  - letter-spacing: -0.03em ~ -0.04em
- **태그**: text-xs ~ sm, font-semibold, tracking 0.15em
- **서브**: text-lg ~ xl, max-w-[28rem]

### Product 비율
- **메인 카드**: min-h 300px → 360px (desktop), max-width ~420px
- **오버랩**: w-[min(220px, 50vw)], 좌하단 -20px ~ -32px 오프셋
- **칩**: 각 4곳 배치, z-index 30~40

### 시각적 깊이
- 메인 카드: `shadow-[0_24px_64px_rgba(45,90,61,0.1)]`, ring-1 ring-white/60
- Back plane: `-inset-6` blur gradient (leaf/blush)
- 칩: `backdrop-blur-md`, `shadow-[0_8px_32px_rgba(0,0,0,0.08)]`

### 칩 위치 (상대 좌표, product wrapper 기준)
- `오늘 1,240 kcal`: top 2%, right -4%
- `단백질 부족`: top 40%, right -2%
- `근처 고단백 맛집 3곳`: bottom 22%, right 4%
- `AI 분석 완료`: bottom 4%, left -6%

### 배경
- `scene-hero-glow` 유지
- 추가: 우측 상단에 lime radial gradient (opacity ~12%)로 제품 영역 강조
