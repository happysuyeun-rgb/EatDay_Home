## B. Redesigned Section Structure

### 원칙

- **카드 없음**: 테두리·그림자 박스로 문장을 감싸지 않음  
- **타이포가 UI**: 크기·색·무게로만 위계  
- **여백이 리듬**: 각 beat 사이에 시각적 숨 고르기  

### 구조 옵션

#### 옵션 1 — Sticky “한 줄 무대” (데스크톱)

- 섹션 전체 `min-height` = (뷰포트 × beat 수) 정도  
- **sticky** 영역: 화면 중앙에 고정된 한 줄 슬롯  
- 스크롤에 따라 **문장만 교체** (이전 문장은 위로 페이드아웃 또는 opacity↓)  
- 장점: 영화적 집중 / 단점: 구현·접근성 고려 필요  

#### 옵션 2 — Semi-sticky + 스크롤 beat (현행 진화)

- 각 beat를 `min-h-[55vh~70vh]` 블록으로 유지  
- 스크롤 시 **한 블록씩** 중앙에 들어오면 해당 문장만 선명하게  
- 이전 beat는 위로 올라가며 opacity 낮춤 (선택)  
- 카드 느낌 제거: **구분선만 아주 얇게** 또는 **완전 무구분**  

#### 옵션 3 — 에디토리얼 컬럼

- 좌측에 얇은 **세로 룰 라인** 또는 **01–04 인덱스** (텍스트만, 카드 아님)  
- 우측에 큰 본문  
- 장점: 잡지 레이아웃 / 단점: 모바일에서 인덱스 재배치 필요  

**추천**: 옵션 2 + 옵션 3의 **가벼운 인덱스**(01–04 또는 짧은 라벨)만 덧대어 템플릿감 완화  

### 배경

- 단색 oat만 반복하지 말고:  
  - 상단: 아주 약한 forest tint (2–4% opacity)  
  - 하단으로 갈수록 blush/lime mist 살짝  
- **미니멀 그래픽**: 큰 블러 orb 1개 이하, 문장 뒤에 흐릿한 숫자 워터마크(예: “1,240”)는 **선택** — 남용 금지  

### 솔루션 전 긴장

- 마지막 beat(4번)에서만 **색·굵기** 살짝 강조 (forest 또는 leaf 진한 톤)  
- 다음 섹션(BeforeAfter)으로 넘어갈 때 **배경 그라데이션**으로 “전환” 암시

## C. Animation Approach

### 진입 (per beat)

- **기본**: `opacity 0→1`, `y: 32~48 → 0`, `blur: 8px → 0` (과하지 않게)  
- **duration**: 0.65–0.85s  
- **easing**: `[0.16, 1, 0.3, 1]`  

### 순차 / 동시

- **Intersection 기반**: 각 beat가 뷰포트 중앙 근처에 올 때 한 번만 재생 (`once: true` 권장)  
- **Sticky 교체 모드**를 쓸 경우: `scrollYProgress`로 문장별 opacity 크로스페이드 — 키프레임을 넓게 잡아 **한 줄이 항상 읽히게**  

### 이전 beat 처리 (선택)

- 스크롤이 지나간 문장: `opacity 0.25~0.4`, `scale 0.98` 정도로 잔상  
- 또는 단순히 위로 스크롤아웃만 (구현 단순)  

### Ambient

- **최소**: 텍스트에 큰 floating 애니메이션은 피함  
- 배경 orb는 `opacity`만 아주 느리게 (20s+) 또는 정적  

### Reduced motion

- blur 제거, y 거리 축소 또는 즉시 `opacity: 1`  
- sticky 교체 → 일반 스택 + 전체 문장 나열

## D. Mobile Simplification

- **min-height**: `min(65vh, 520px)` 수준으로 다소 축소 — 한 화면에 반쯤 다음 beat가 보이면 스크롤 동기 유도  
- **타이포**: `text-2xl` 하한 유지, `max-w-[20ch]`로 줄 길이 제한  
- **인덱스 01–04**: 모바일에서는 문장 위 **한 줄**로만 표시하거나 숨김  
- **border-b**: 모바일에서는 제거하거나 마지막 beat 전만 유지  
- **Sticky 복잡 모드**: 모바일에서는 **비활성화**하고 옵션 2 스택만 사용 권장

## E. Implementation Recommendations

### 타이포 스케일 (참고)

- 기본 beat: `text-2xl sm:text-3xl md:text-4xl lg:text-[2.75rem]`  
- 강조 beat(4번): `font-semibold`, `text-[var(--accent-forest)]`, 한 단계 크기 업  

### 레이아웃

- `max-w-4xl` ~ `max-w-5xl`, 좌 정렬 또는 중앙 정렬 중 하나로 통일 (에디토리얼은 **좌 정렬**이 종종 더 강함)  
- `leading`: 1.2–1.35  
- `tracking`: `-0.02em`  

### 기술 스택

- Framer Motion: `useInView` + `motion.p` (현행과 호환)  
- Sticky 교체: `useScroll` + `useTransform` — 이전 이슈(빈 화면) 방지를 위해 **offset `["start end", "end start"]`** 및 키프레임 검증 필수  
- **접근성**: 각 beat를 `aria-live="polite"`로 묶는 것은 스크롤 시 과도한 알림이 될 수 있음 → **sr-only 요약 h2** 유지  

### 다음 섹션 연결

- Empathy 마지막 줄과 BeforeAfter 첫 헤드라인이 **논리적으로 이어지게** 카피 조정 검토  
  - 예: Empathy 끝 “선택의 순간엔 혼자” → BeforeAfter “같은 목표, 다른 방식”  

### 체크리스트

- [ ] 카드·박스 UI 없음  
- [ ] 4 pain을 4문장(또는 4+1)으로 명확히 매핑  
- [ ] 큰 타이포 + 넉넉한 세로 여백  
- [ ] 순차 reveal (in-view 또는 sticky)  
- [ ] 마지막 beat에서만 시각적 긴장 상승  
- [ ] 모바일: 높이·sticky 단순화  
- [ ] `prefers-reduced-motion` 대응
